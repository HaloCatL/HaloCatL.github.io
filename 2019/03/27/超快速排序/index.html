超快速排序

发表于 2019-03-27 | 分类于 OI
字数统计: 1.9k | 阅读时长 ≈ 7

在这个问题中，您必须分析特定的排序算法——超快速排序。

该算法通过交换两个相邻的序列元素来处理n个不同整数的序列，直到序列按升序排序。

对于输入序列9 1 0 5 4，超快速排序生成输出0 1 4 5 9。

您的任务是确定超快速排序需要执行多少交换操作才能对给定的输入序列进行排序。

输入格式

输入包括一些测试用例。

每个测试用例的第一行输入整数$n$，代表该用例中输入序列的长度。

接下来n行每行输入一个整数$a_i$,代表用例中输入序列的具体数据，第$i$行的数据代表序列中第i个数。

当输入用例中包含的输入序列长度为0时，输入终止，该序列无需处理。

输出格式

对于每个需要处理的输入序列，输出一个整数op，代表对给定输入序列进行排序所需的最小交换操作数，每个整数占一行。

数据范围

$0≤N<500000,$
$0≤a_i≤999999999$

这个题可以转换模型来解答。通过题目我们可以了解，这个题目其实就模拟了一遍冒泡排序的过程，而冒泡排序的交换次数就是序列中逆序对的个数，知道这个了之后，我们就可通过求整个序列的逆序对个数来得到我们排序完成所需的交换次数。

这里需要了解一个新概念，逆序对。
对于一组数列，对于数列中的任意两个元素$a_i, a_j$，若$i < j, a_i > a_j$，那么就说明$a_i, a_j$为一对逆序对。

如果快速求我们的逆序对，简单的方法有归并排序与树状数组与线段树，对于前者就是我们这次要讲解的内容，而后两者..等我学了再来填坑⑧..

归并排序的过程很简单，对于一个数列，我们每次从中间断开（类似二分的思想），然后对于左右两个序列继续进行这个操作，直到只剩一个为止，然后开始每个区间排序（此时每个区间已经是有序的序列（为什么直接是有序数列了？因为我们是从底层开始计算的，刚开始只有一个元素的时候就已经是有序序列了，那么根据下方算法合并之后我们仍然是一个有序序列），我们仅是需要让两个区间合为一个区间时进行计算），那么这样排序会有三种情况

①：左边单独排，这样我们每次只用判断$a_i, a_{i+1}$的关系即可
②：右边单独排，这样我们每次也只用判断$a_i, a_{i + 1}$的关系
③：左右一起排，这样我们每次需要判断$a_i, a_j$的关系

很明显，对于这三种方案，我们先进行第三个然后再进行第一第二个就能最优减少执行数。到这里，我们可以考虑如何求得逆序对了，首先对于逆序对的话，我们只会在第三种情况中出现（为什么？因为我们先执行的③ 如果我们③执行完了开始执行①②的时候，我们是不会出现位与位的交换（此时只会执行①或②，在③中我们已经至少处理了一个完整数列了），所以不存在逆序对）

知道这个之后，我们来看看如何求这个逆序对个数。首先我们设当前有一个数列$a_x$，它分成了两个数列 分别为$a_1$与$a_2$，此时我们在$a_1$中遍历到了$i$位，在$a_2$中遍历到了$j$位，再设这个数列$a_x$的中间位置的编号为$mid$，那么显然可得$i < mid$ 与 $j >= (mid + 1)$ 此时，如果第$i$位 $<=$ 第$j$位，此时不发生交换，那么我们这种情况不算入逆序对中；如果第$i$位 $>$ 第$j$位，那么说明此时要发生交换，因为此时我们的序列是有序序列，那么说明从第$i$位到第$mid$位上的数都能与第$j$位成立逆序关系，所以这次交换造成的总逆序对个数就是$mid - i + 1$（为什么+1？在草稿上随便列几个数字看看？）
