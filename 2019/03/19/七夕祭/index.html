七夕祭

发表于 2019-03-19 | 分类于 OI
字数统计: 1.7k | 阅读时长 ≈ 6

七夕节因牛郎织女的传说而被扣上了「情人节」的帽子。

于是TYVJ今年举办了一次线下七夕祭。

Vani同学今年成功邀请到了cl同学陪他来共度七夕，于是他们决定去TYVJ七夕祭游玩。

TYVJ七夕祭和11区的夏祭的形式很像。

矩形的祭典会场由N排M列共计$N×M$个摊点组成。

虽然摊点种类繁多，不过cl只对其中的一部分摊点感兴趣，比如章鱼烧、苹果糖、棉花糖、射的屋……什么的。

Vani预先联系了七夕祭的负责人zhq，希望能够通过恰当地布置会场，使得各行中cl感兴趣的摊点数一样多，并且各列中cl感兴趣的摊点数也一样多。

不过zhq告诉Vani，摊点已经随意布置完毕了，如果想满足cl的要求，唯一的调整方式就是交换两个相邻的摊点。

两个摊点相邻，当且仅当他们处在同一行或者同一列的相邻位置上。

由于zhq率领的TYVJ开发小组成功地扭曲了空间，每一行或每一列的第一个位置和最后一个位置也算作相邻。

现在Vani想知道他的两个要求最多能满足多少个。

在此前提下，至少需要交换多少次摊点。

输入格式

第一行包含三个整数$N$和$M$和$T$，$T$表示$c$l对多少个摊点感兴趣。

接下来$T$行，每行两个整数$x, y$，表示$cl$对处在第$x$行第$y$列的摊点感兴趣。

输出格式

首先输出一个字符串。

如果能满足Vani的全部两个要求，输出both；

如果通过调整只能使得各行中cl感兴趣的摊点数一样多，输出row；

如果只能使各列中cl感兴趣的摊点数一样多，输出column；

如果均不能满足，输出impossible。

如果输出的字符串不是impossible， 接下来输出最小交换次数，与字符串之间用一个空格隔开。

数据范围

$1≤N,M≤100000$
$0≤T≤min(N×M,100000)$
$1≤x≤N$
$1≤y≤M$

题目题解：

这道题就相当于均分纸牌，但是题面有不同的地方，对于这道题，我们先考虑只有行或者只有列，那么我们可以发现，这个移动还可以是首尾互相交换的，那么我们整个数列就是一个环，对于环我们就得用其他的方法来解决了，用原来的方法肯定是行不通的。

难题我们就从一个简单的模型中化简，从而解决我们一个很难的模型。
先继续考虑均分纸牌，因为这道题的部分情况只是在均分纸牌的基础上加了一个环罢了，然后我们来考虑均分纸牌，对于均分纸牌这道题我们做一些修改，我们不能一次性移动多个纸牌，最多只能由自己转移一张纸牌或者从别的牌堆那里转移一张纸牌，那么我们就能得到一个答案通式 $\sum ^n_{i = 1}\left |i\times \frac{T}{n} - G_i \right |$，$T$为总牌数，$G_i$是$a_i$的前缀和，$a_i$为编号$i$的牌堆的牌数，我们利用这个通式就能算出答案（对于链型）

我们简单的解决了链型的均分纸牌，我们看看能不能解决环形的问题。对于环，我们有一个很经典的环断为链的形式，就是考虑从第$k$个堆断开，写成一行，这$n$个牌堆持有的纸牌数的前缀和我们便能写出来了，即第$k + 1$位是$f_{k + 1} - f_k $ （这里$f_i$就是在一个数列内编号为$i$的前缀和），那么第$k + 2$位就是$f_{k + 2} - f_k$，那么第$1$位就是$f_1 + f_n - f_k$，这里$n$就是我们总共的牌堆数。

根据这样我们不管在哪里断环，我们都能很简单的得到当前链每个数的前缀和，根据这个变式，我们还能进行更好的优化，即每个牌堆的牌数在加到前缀和里之前就提前减去一个$\frac{T}{n}$，那么这样我们就能得到一个每个牌堆仅存了一个当前牌数比平均数少（多）的数了，这样很显然，对我们接下来需要判断的答案不造成影响，而且如果我们有需要用到原数的地方，直接加上$\frac{T}{n}$就好了。当然，这样处理之后，我们就可以继续化简我们上面谈到的$f_1 + f_n - f_k$，因为我们既然有一个合法的方案，那么一定有$f_n = 0$（不懂请重新读一遍文字），那么我们就能化简为$\sum ^n_{i = 1}\left |f_i - f_k \right |$，那么很显然，我们只用让这里面的式子最小就可以达到我们的目的了，如何让这个等式最小？ 你是否还没忘掉货仓选址？

很显然，利用中位数就能很快的得出答案了。

好了，我们对于单一行或单一列的答案已经求出来，此时我们需要求同时一行和一列，如何解决？ 因为我们一次移动两个数，仅仅会影响其单一行或单一列的目标，而不会同时影响一行和一列，所以我们单独处理两个。如果合法，那么就两个答案加起来，如果单个合法，那么就处理单个就好了。
